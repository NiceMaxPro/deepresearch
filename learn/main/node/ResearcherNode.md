# ResearcherNode

该节点通过执行网络搜索并通过 LLM 生成研究摘要来执行计划中的研究类型步骤。它找到分配给它的第一个未完成的 `RESEARCH` 步骤，处理反思逻辑以重新尝试之前的失败，并构建一个包含步骤详情和任何反思历史的任务消息。该节点通过 `SearchInfoService` 使用配置的搜索引擎执行搜索，将搜索结果追加到消息列表中，并调用一个 AI 代理（可选配 MCP 工具回调，以及可路由到专用代理的智能代理选择）。它添加了一个引用提醒，指示 LLM 避免使用内联引用，而是在末尾使用链接引用格式。结果以流式方式传输并存储在 `researcher_content_<executorNodeId>` 下，搜索结果则累积在 `site_information` 中。
