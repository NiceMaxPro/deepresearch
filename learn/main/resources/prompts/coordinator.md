# coordinator.md

此提示模板定义了协调者（coordinator）代理，其自我介绍为"阿里巴巴图深度研究助手"。它的主要角色是处理问候和闲聊，同时将所有实质性研究任务移交给专业的规划者（planner）代理。提示定义了三类请求：直接处理（问候、闲聊、能力询问）、礼貌拒绝（提示泄露、有害内容、冒充、安全绕过尝试）以及移交给规划者（所有事实性问题、研究请求、分析任务）。

对于第 1 类，代理以纯文本问候回应。对于第 2 类，代理以礼貌拒绝回应。对于第 3 类——涵盖大多数用户输入——代理调用 `handoff_to_planner()` 而不附加额外推理。代理必须在相关时始终自报名称，保持友好但专业的语气，接受任何语言的输入并以相同语言回应，并在不确定如何处理请求时默认移交给规划者。
