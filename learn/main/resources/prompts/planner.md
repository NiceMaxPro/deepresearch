# planner.md

此提示模板定义了规划者（planner）代理，一位专业的深度研究者，负责编排研究团队以收集全面的信息。它建立了严格的信息质量标准：全面覆盖主题的所有方面和多个视角、足够的深度（包含详细数据点和统计数据）以及超出最低要求的足够数量。上下文评估框架评估现有信息是否充分，采用非常严格的标准，除非满足所有充分性条件，否则默认收集更多数据。

规划者使用一个八维分析框架来分解主题：历史背景、当前状态、未来指标、利益相关者数据、定量数据、定性数据、比较数据和风险数据。步骤被分类为研究步骤（需要网页搜索）或处理步骤（API 调用、计算、数据库查询）。输出是一个原始 JSON `Plan` 对象，包含 `has_enough_context`、`thought`、`title` 以及一个 `Step` 对象数组，每个步骤包含 `need_web_search`、`title`、`description` 和 `step_type` 字段。步骤数量受 `{{ max_step_num }}` 模板变量限制。提示明确禁止汇总或合并步骤。
