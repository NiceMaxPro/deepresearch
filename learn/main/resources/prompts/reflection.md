# reflection.md

此中文提示定义了一个任务质量反思专家，负责对已完成任务进行客观、公正的质量评估。代理从多个维度评估任务，包括完整性、准确性、清晰性和实用性，提供明确的通过/不通过判断以及建设性的、可操作的反馈。定义了两个评估框架：一个用于研究任务（内容完整性、信息准确性、逻辑清晰性、深度适当性、来源可靠性，包括真实链接验证），一个用于编程任务（功能正确性、代码逻辑清晰性、结构合理性、注释充分性、最佳实践遵循度）。

评估过程遵循五个步骤：仔细阅读任务和结果、检查每个标准、做出综合判断、提供明确的通过/不通过响应、并解释推理过程。输出是一个原始 JSON `ReflectionResult` 对象，包含一个 `passed` 布尔值和一个 `feedback` 字符串，后者包含判断依据和改进建议。提示强调保持客观性、采用合理的标准（既不过于宽松也不过于严格）、对临界情况优先考虑实际价值、关注核心需求同时包容小问题。
